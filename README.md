---

## 5. Принцип работы

Система не генерирует контекст автоматически.  
Она читает только то, что лежит в `context/`.

Если файл отсутствует — его не существует для модели.  
Если данные не добавлены — модель их не видит.

Таким образом:
- нет случайных инжектов  
- нет скрытых системных промптов  
- нет “магического” поведения  

---

## 6. Наблюдаемость

Каждое действие агента отслеживается через :contentReference[oaicite:0]{index=0}.

Фиксируется:
- входной контекст  
- вызовы модели  
- использование инструментов  
- количество токенов  
- стоимость  

Это даёт полное понимание того, что происходит внутри.

---

## 7. Модель агента

Агент — это простой цикл:

1. чтение контекста из файлов  
2. формирование запроса  
3. вызов модели  
4. обработка результата  
5. обновление состояния (если нужно)  

Без скрытых шагов.

---

## 8. Суть

coreclaw — это система, где:

- контекст полностью под контролем  
- структура данных прозрачна  
- каждое действие можно отследить  
- поведение модели объяснимо  

Основной фокус — не в “интеллекте” агента, а в том,  
что именно он видит и как с этим работает.
